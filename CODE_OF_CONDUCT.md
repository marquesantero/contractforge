# Code Of Conduct

ContractForge contributors are expected to keep technical discussions direct,
respectful and focused on the work.

## Expected Behavior

- Be precise when reporting bugs or reviewing changes.
- Challenge designs with concrete reasoning and evidence.
- Assume good intent, but ask for clarification when intent is unclear.
- Keep security-sensitive information private.
- Respect maintainers' decisions about scope, package boundaries and release
  timing.

## Unacceptable Behavior

- Harassment, threats, personal attacks or discriminatory language.
- Publishing private credentials, tenant details or account data.
- Repeatedly derailing issues or pull requests after a maintainer has clarified
  scope.
- Misrepresenting test results, maturity status or platform support.

## Enforcement

Maintainers may edit, hide or remove comments, close issues, reject pull
requests, block users or report abuse to GitHub when needed to protect the
project and its contributors.
