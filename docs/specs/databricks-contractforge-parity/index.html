Databricks ContractForge Parity

Purpose​

The Databricks adapter should not lose mature Databricks adapter capabilities from the Databricks adapter baseline.

The target is not a byte-for-byte copy. The target is equivalent behavior and evidence:

  • same ContractForge contract semantics
  • same Databricks execution capabilities where they are still Databricks-owned
  • same control/evidence discipline
  • no Databricks/Spark dependency leaking into contractforge_core

Parity Status​

StatusMeaning
PORTEDEquivalent behavior exists in contractforge_databricks.
PORTED_IN_COREEquivalent behavior exists in the platform-neutral ContractForge core/CLI and should not move into the Databricks adapter.
PORTED_AS_REVIEW_ARTIFACTThe adapter renders auditable artifacts, but execution remains runtime/deployment-owned.
PORT_IN_PROGRESSPartially ported; known work remains.
INTENTIONALLY_ADAPTER_OWNEDCapability exists only in Databricks adapter and must not move to core.
NOT_PORTED_YETmature Databricks capability still needs adapter implementation.

Technical Row Columns​

The Databricks adapter preparation layer added technical columns to target rows:

  • ingestion_date
  • ingestion_ts_utc
  • source_system
  • __run_id

In ContractForge Core these are not mandatory semantic columns. The durable audit source of truth is the evidence model and Databricks control tables. In particular, ingestion_date is treated as a legacy physical partitioning convenience from the Databricks adapter baseline, not as a required ContractForge contract concept.

Databricks presets and runtime behavior must not assume ingestion_date exists by default. If a deployment wants physical row metadata columns, the Databricks adapter should expose that as an explicit adapter-owned option rather than silently adding them from the core.

Spark utility conveniencesPORTEDruntime/spark.py ports safe_cache, safe_unpersist, table_exists, schema_signature, fix_encoding, runtime info and DataFrame-level Delta schema sync as Databricks adapter runtime helpers.
SQL/runtime utility conveniencesPORTEDsql/ and runtime/utils.py port SQL literals, JSON literal serialization, UTC/run-id helpers, truncation, list parsing and column validation without requiring PySpark imports.
Unity Catalog capability issuesPORTEDcapabilities/uc.py ports conservative issue reporting for comments, tags, grants, row filters and masks when the target/capability evidence cannot safely support the declared governance feature.

Non-Negotiable Rule​

If a Databricks capability existed in the Databricks adapter baseline and is not ported yet, it must appear in this matrix as NOT_PORTED_YET or PORT_IN_PROGRESS.

No capability should be silently dropped just because the core is now platform-neutral.
