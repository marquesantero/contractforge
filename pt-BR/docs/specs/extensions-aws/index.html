AWS Extensions Specification

Propósito​

extensions.aws é o único local do contrato onde os parâmetros de execução de propriedade do AWS podem aparecer.

O núcleo valida extensions como um mapa opaco. O adaptador AWS possui o significado, as verificações de compatibilidade, a renderização e o comportamento de tempo de execução das chaves em extensions.aws. Essas chaves não devem se tornar campos principais de nível superior e não devem ser lidas por outros adaptadores.

Canonical Shape​

Declare apenas campos de extensão AWS que substituem os padrões do adaptador ou são necessários para um contrato específico. O exemplo abaixo mostra diversas substituições opcionais; isto não é a configuração AWS mínima necessária.

extensions:
  aws:
    glue_job:
      name: cf-orders
      role_arn: arn:aws:iam::123456789012:role/ContractForgeGlueRole
      script_s3_uri: s3://contractforge-artifacts/dev/orders/orders.glue_job.py
      worker_type: G.1X
      number_of_workers: 4
      timeout_minutes: 60
      max_retries: 0
      default_arguments:
        "--TempDir": s3://contractforge-artifacts/tmp/
    iceberg:
      warehouse: s3://company-lakehouse/warehouse/
      table_properties:
        write.format.default: parquet
    job_bookmarks:
      enabled: true

Se esses campos forem omitidos, o adaptador AWS padronizará a versão Glue 4.0, tipo de trabalhador G.1X, dois trabalhadores, tempo limite de 60 minutos, zero tentativas e modo executor de biblioteca. A ativação do marcador é inferida a partir da semântica de origem a menos que job_bookmarks ou glue_job.enable_job_bookmark seja declarado.

Chaves suportadas​

KeyTypeComportamento do adaptador
glue_jobmapAWS Glue botões de renderização e implantação de trabalho, como nome do trabalho, função ARN, script S3 URI, formato do trabalhador, tempo limite, novas tentativas e argumentos de trabalho padrão.
icebergmapIceberg/Glue Opções de tabela de catálogo que não pertencem à semântica de destino portátil. warehouse define spark.sql.catalog.glue_catalog.warehouse para tarefas Glue Spark e deve ser um s3:// URI. table_properties são aplicados somente em caminhos create/createOrReplace gerados por Iceberg.
lake_formationmapOpções de aplicação/revisão do Lake Formation para concessões, filtros de linha e artefatos de coluna/segurança.
job_bookmarksmapAWS Ajuste de marcador Glue para contratos incrementais incremental_files ou JDBC portáteis.
dqdlmapAWS Glue Preferências de renderização/tempo de execução de qualidade de dados.
dependenciesmapDicas de dependência de tempo de execução, como módulos Python extras ou jars de conector.
native_passthroughmapOpções de transferência de conector nativo AWS para conectores nativos/personalizados AppFlow, DMS, Glue ou serviços relacionados.

glue_job.default_arguments é apenas aditivo. Ele não deve substituir argumentos Glue de propriedade do adaptador, como --datalake-formats, --enable-glue-datacatalog, --job-language, --job-bookmark-option ou argumentos de dependência renderizados pelo adaptador (--additional-python-modules, --extra-jars, --extra-py-files). O adaptador aumenta em vez de enfraquecer silenciosamente o tempo de execução Glue gerado.

Os campos aninhados consumidos também estão na lista de permissões. Campos aninhados com erros ortográficos avisam com AWS_UNKNOWN_EXTENSION_FIELD e são ignorados:

  • glue_job: name, role_arn, script_s3_uri, glue_version, worker_type, number_of_workers, timeout_minutes, max_retries, default_arguments, description, enable_job_bookmark.
  • iceberg: warehouse, table_properties.
  • job_bookmarks: enabled, enable_job_bookmark.
  • dependencies: python_modules, additional_python_modules, jars, extra_jars, py_files, extra_py_files.

O adaptador pode suportar chaves adicionais em versões secundárias, mas cada nova chave requer:

  1. uma atualização para esta especificação;
  2. um teste de planejamento/renderização/tempo de execução;
  3. comportamento documentado quando o tempo de execução AWS não consegue preservar a semântica;
  4. nenhuma importação ou mudança de comportamento em contractforge_core.

Comportamento de validação​

O núcleo:

  • accepts extensions as an opaque map;
  • não interpreta chaves de extensão AWS;
  • não importa contractforge_aws.

O adaptador AWS:

  • reads only extensions.aws;
  • avisa sobre chaves desconhecidas com AWS_UNKNOWN_EXTENSION;
  • avisa sobre chaves reconhecidas com formato errado usando AWS_EXTENSION_SHAPE_IGNORED;
  • never honors unknown extension keys silently;
  • never honors malformed extension values silently;
  • não deve permitir extensões de contrato para relaxar os controles de segurança, como proteção SSRF ou rejeição de segredo em linha.

Regra de limite​

Prefira parâmetros principais portáteis em vez de extensões AWS. Se um recurso tiver intenção equivalente em todas as plataformas, mas AWS precisar de detalhes extras, adicione um parâmetro semântico central opcional com nomenclatura neutra de plataforma. Use extensions.aws somente quando o comportamento for genuinamente específico do AWS.
