Especificação do Contrato Ambiental

Propósito​

O contrato de ambiente descreve onde e como um adaptador pode executar, renderizar, publicar evidências e resolver infraestrutura.

Não é um contrato de ingestão semântica. A alteração do environment.yaml não deve alterar quais dados são ingeridos, qual modo de gravação é solicitado, quais anotações existem, quais metadados de operações são declarados ou qual política de acesso se destina.

File Name​

Recommended file name:

environment.yaml

Shape​

environment:
  name: prod
  adapter: databricks

  runtime:
    kind: serverless
    warehouse_id: ${DATABRICKS_WAREHOUSE_ID}

  deployment:
    artifact: databricks_asset_bundle
    workspace_path: /Workspace/ContractForge

  artifacts:
    uri: s3://contractforge-artifacts/prod/orders/
    include_contract_bundle: true
    include_normalized_contract: true

  evidence:
    catalog: main
    schema: contractforge_ops

  secrets:
    strategy: secret_scope
    scope: prod-secrets

  defaults:
    timezone: UTC
    fail_fast: true

  capabilities:
    require:
      - merge
      - evidence_store

  parameters:
    databricks:
      cloudFiles.inferColumnTypes: true
      job.max_concurrent_runs: 1

Fields​

FieldOwnerPropósito
nameEssencialNome do ambiente lógico, como dev, qa, prod ou client-a-prod.
adapterEssencialSeletor de adaptador, como databricks, aws, gcp, snowflake ou fabric.
runtimeAdaptadorContexto de computação/tempo de execução.
deploymentAdaptadorContexto de artefato e implantação.
artifactsAdaptadorLocal de publicação de artefato e opções de retenção. AWS usa isso para scripts Glue renderizados, manifestos e artefatos de contrato originais/normalizados em S3.
evidenceAdaptadorLocalização do armazenamento de evidências e contexto de persistência.
secretsAdaptadorSecret resolution strategy.
defaultsAdaptadorPadrões de execução em nível de ambiente.
capabilitiesNúcleo + adaptadorNomes de recursos obrigatórios, preferenciais ou proibidos.
parameters.<adapter>AdaptadorParâmetros do adaptador nativo.

Fabric Secret Bindings​

Fabric authenticated HTTP/REST notebook drafts resolve Espaços reservados {{ secret:scope/key }} por meio do Azure Key Vault em tempo de execução. O contrato mantém apenas o espaço reservado; o ambiente vincula o espaço reservado scope to a vault URL:

environment:
  adapter: fabric
  secrets:
    vault_url: https://contractforge-default.vault.azure.net/
    scopes:
      fabric: https://contractforge-fabric.vault.azure.net/

Generated notebooks call notebookutils.credentials.getSecret(vault_url, key). A identidade de tempo de execução Fabric deve ter permissão para ler o Key Vault declarado segredo. Tokens de portador inline, chaves API e senhas permanecem fora do stable limite do contrato.

Defaults​

environment.name é padronizado como dev quando omitido. O seletor de adaptador permanece obrigatório porque escolhe o limite da plataforma.

Os botões de tempo de execução do adaptador devem ser padrão no adaptador quando o padrão for estável e documentado. Para a política padrão mais ampla, consulte Parameter defaults.

Forbidden Fields​

O contrato ambiental não deve conter campos de contrato semântico:

  • source
  • target
  • target_table
  • mode
  • merge_keys
  • schema_policy
  • quality_rules
  • annotations
  • operations
  • access
  • transform

Se um campo alterar a intenção de ingestão, ele pertencerá à ingestão, às anotações, às operações ou ao contrato de acesso.

Limite Central​

O núcleo valida a forma genérica do ambiente e rejeita campos semânticos.

O núcleo não interpreta chaves específicas da plataforma:

  • runtime
  • deployment
  • artifacts
  • evidence
  • secrets
  • defaults
  • parameters

Os adaptadores possuem esses significados.

Regra de vinculação do adaptador​

Um adaptador pode usar valores de ambiente para escolher tempo de execução, implementação, localização de evidências e padrões nativos.

Um adaptador não deve usar valores de ambiente para substituir campos de contrato semântico, como origem, destino, modo de gravação, anotações, operações ou acesso.

AWS Artifact Publication​

AWS As tarefas Glue não podem ler arquivos YAML locais de uma estação de trabalho do desenvolvedor. O AWS portanto, o adaptador renderiza artefatos de tempo de execução localmente ou em CI, publica-os em o prefixo artifacts.uri S3, materializa a definição de tarefa Glue com o publicou o script URI e registra a tarefa nativa Glue.

Esta é uma preocupação ambiental porque descreve onde os artefatos do adaptador estão armazenado, não o que a ingestão faz.
