GeoJSON E2E Test Presentation Runbook

Propósito​

Este runbook apresenta a prova de paridade do adaptador ContractForge com base no Contratos de medalhão USGS GeoJSON. Mantenha a página de documentação pública chamada test-contracts-across-adapters; essa página é o canônico lado a lado comparação de contratos para Databricks, AWS, Snowflake, Fabric e GCP.

O cenário prova que a mesma intenção de ingestão pode ser executada em todos os ambientes estáveis adaptadores com apenas tabela nativa, dialeto SQL, armazenamento e rede de contas bindings changed:

  • public USGS 2.5_day.geojson earthquake payload;
  • bronze raw GeoJSON landing;
  • modelagem de evento de prata de features[];
  • extração de coordenadas, normalização de carimbo de data/hora e desduplicação;
  • agregados diários de ouro e por faixa de magnitude;
  • execução nativa do adaptador e evidência da tabela de controle.

A regra de apresentação é estrita: nenhum código de projeto alternativo. Contratos, arquivos de ambiente e CLIs do adaptador conduzem a execução.

Roteiro da apresentação​

  1. Open test-contracts-across-adapters.
  2. Mostre que a intenção do contrato é a mesma: carga útil de origem, bronze bruto pouso, formato do evento de prata, agregados de ouro e regras de qualidade.
  3. Mostre as pequenas diferenças de ligação nativa para cada adaptador.
  4. Execute ou apresente as execuções do adaptador.
  5. Compare as contagens de linhas de destino e as tabelas de evidências.
  6. Chame a portabilidade honesta: onde uma ligação de origem não é nativa, o O adaptador deve usar vinculação explícita necessária para revisão ou específica do adaptador, e não uma hidden workaround.

Commands To Reproduce​

O planejamento e a renderização local devem usar diretamente os contratos do projeto:

uv run contractforge validate-project examples/real-world/usgs-earthquake-rest-medallion
uv run contractforge-databricks render-project-bundle examples/real-world/usgs-earthquake-rest-medallion/project.yaml
uv run contractforge-aws deploy-project examples/real-world/usgs-earthquake-rest-medallion/project.yaml --dry-run --summary-only
uv run contractforge-snowflake deploy-project examples/real-world/usgs-earthquake-rest-medallion/project.yaml --dry-run --summary-only
uv run contractforge-fabric run-project examples/real-world/usgs-earthquake-rest-medallion/project.yaml --environment-key fabric
uv run contractforge-gcp run-project examples/real-world/usgs-earthquake-rest-medallion/project.yaml --environment-key gcp

Os comandos de tempo de execução dependem dos perfis e ambientes configurados, mas o O portão de aceitação é o mesmo para todos os adaptadores: os quatro alvos medalhões devem completo a partir de contratos e evidências devem mostrar execuções bem-sucedidas, qualidade de aprovação, sem linhas de quarentena e sem erros.

Claims Allowed​

  • GeoJSON é a prova de paridade entre adaptadores para Databricks, AWS, Snowflake, Fabric e GCP.
  • A mesma intenção do medalhão é preservada em Delta, Iceberg, Snowflake tabelas nativas e tabelas nativas BigQuery.
  • Os artefatos de tempo de execução são nativos de cada plataforma.
  • A evidência da tabela de controle é uma superfície do produto ContractForge.
  • As vinculações de origem e de tabela podem ser diferentes quando são contratos explícitos. parameters.
  • Snowflake A execução de REST tem um pré-requisito de conta explícito: CF_USGS_REST_ACCESS deve permitir HTTPS de saída para earthquake.usgs.gov:443.

Claims To Avoid​

  • Não afirme que todos os conectores de origem são igualmente nativos em todos os adaptadores.
  • Não use testes JDBC/database-source como prova de paridade de três adaptadores.
  • Não reivindique que recursos não suportados sejam traduzidos silenciosamente.
  • Não reivindique semântica histórica ou comportamento de governança além da maturidade evidências já registradas para cada adaptador.

Cleanup​

Limpe somente depois que as evidências de liberação forem capturadas.

Databricks cleanup:

DROP SCHEMA IF EXISTS workspace.cf_usgs_rest_bronze CASCADE;
DROP SCHEMA IF EXISTS workspace.cf_usgs_rest_silver CASCADE;
DROP SCHEMA IF EXISTS workspace.cf_usgs_rest_gold CASCADE;
DROP SCHEMA IF EXISTS workspace.cf_usgs_rest_ops CASCADE;

AWS cleanup:

aws glue get-jobs --region $AWS_REGION
aws glue delete-job --job-name <job-name> --region $AWS_REGION
aws glue delete-database --name contractforge_cf_usgs_rest_bronze --region $AWS_REGION
aws glue delete-database --name contractforge_cf_usgs_rest_silver --region $AWS_REGION
aws glue delete-database --name contractforge_cf_usgs_rest_gold --region $AWS_REGION
aws s3 rm s3://$AWS_ARTIFACT_BUCKET/contractforge-usgs-rest/ --recursive --region $AWS_REGION

Snowflake cleanup:

DROP TABLE IF EXISTS CONTRACTFORGE_TEST_DB.PUBLIC.CF_USGS_REST_BRONZE;
DROP TABLE IF EXISTS CONTRACTFORGE_TEST_DB.PUBLIC.CF_USGS_REST_SILVER;
DROP TABLE IF EXISTS CONTRACTFORGE_TEST_DB.PUBLIC.CF_USGS_REST_GOLD_DAILY;
DROP TABLE IF EXISTS CONTRACTFORGE_TEST_DB.PUBLIC.CF_USGS_REST_GOLD_BANDS;
