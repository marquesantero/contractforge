Operar a ingestão por meio de evidências de tabela de controle.

ContractForge grava evidências operacionais em tabelas de controle Delta para que as equipes possam diagnosticar falhas, rastrear atualizações, auditar governança, inspecionar lotes de streaming, estimar o custo do runtime e verificar versões da estrutura sem depender da saída do notebook.

Modelo de esquema de controle​

As tabelas de controle são criadas por ensure_ctrl_tables() no catálogo de controle e esquema configurados. Eles são aditivos: as atualizações da estrutura adicionam colunas conhecidas quando ausentes e não removem colunas existentes automaticamente.

Nomes padrão

Nomes de tabelas estáveis​

O prefixo padrão é ctrl_ingestion_*. Os nomes podem ser alterados por meio de FrameworkConfig, mas os painéis devem preferencialmente resolver nomes do esquema de controle configurado.

Versionamento

Estrutura e esquema de controle​

A maioria das tabelas operacionais inclui framework_version e ctrl_schema_version. Use-os para detectar versões mistas e layouts de metadados desatualizados.

Partições

Tabelas de alto volume particionadas por data​

ctrl_ingestion_runs e ctrl_ingestion_errors são particionados por data. Use filtros de data em painéis e trabalhos de retenção.

Cargas JSON

Detalhes operacionais flexíveis​

Opções de conector, metadados de origem, durações de estágio, alterações de esquema e métricas de operação são armazenados como strings JSON para manter o esquema da tabela estável.

Mapa de tabela​

Referência de campo​

Use esta seção como dicionário de dados operacionais para painéis, regras de alerta e consultas de suporte. Os campos JSON são armazenados como strings; analise-os com funções from_json, get_json_object ou SQL JSON ao criar visualizações operacionais selecionadas.

Consultas SQL úteis​

Substitua main.ops pelo catálogo e esquema usados ​​para tabelas de controle.

Integridade da execução recente​

SELECT
  status,
  target_table,
  mode,
  source_connector,
  runtime_type,
  rows_read,
  rows_written,
  rows_quarantined,
  duration_seconds,
  error_message,
  run_id
FROM main.ops.ctrl_ingestion_runs
WHERE run_date >= current_date() - INTERVAL 7 DAYS
ORDER BY started_at_utc DESC
LIMIT 100;

Triagem de falhas com full stack trace​

SELECT
  r.started_at_utc,
  r.target_table,
  r.source_connector,
  r.mode,
  r.runtime_type,
  r.error_message AS short_error,
  e.error_type,
  e.stack_trace
FROM main.ops.ctrl_ingestion_runs r
LEFT JOIN main.ops.ctrl_ingestion_errors e
  ON r.run_id = e.run_id
WHERE r.status = 'FAILED'
  AND r.run_date >= current_date() - INTERVAL 7 DAYS
ORDER BY r.started_at_utc DESC;

Frescura e desvio de marca d'água​

SELECT
  target_table,
  watermark_column,
  watermark_value,
  last_success_at_utc,
  last_status,
  last_rows_written,
  last_delta_version,
  current_timestamp() - last_success_at_utc AS time_since_success
FROM main.ops.ctrl_ingestion_state
ORDER BY last_success_at_utc ASC NULLS FIRST;

Regras de qualidade que requerem atenção​

SELECT
  q.target_table,
  q.rule_name,
  q.severity,
  q.status,
  q.failed_count,
  q.message,
  r.run_id,
  r.started_at_utc
FROM main.ops.ctrl_ingestion_quality q
JOIN main.ops.ctrl_ingestion_runs r
  ON q.run_id = r.run_id
WHERE r.run_date >= current_date() - INTERVAL 14 DAYS
  AND q.failed_count > 0
ORDER BY r.started_at_utc DESC, q.failed_count DESC;

Volume de quarentena por destino e regra​

SELECT
  target_table,
  rule_name,
  count(*) AS quarantined_records,
  max(quarantined_at_utc) AS latest_quarantine
FROM main.ops.ctrl_ingestion_quarantine
GROUP BY target_table, rule_name
ORDER BY quarantined_records DESC;

Mudanças de esquema por tabela​

SELECT
  target_table,
  change_type,
  column_name,
  source_type,
  target_type,
  applied,
  change_ts_utc,
  details_json
FROM main.ops.ctrl_ingestion_schema_changes
ORDER BY change_ts_utc DESC;

Streaming agregado versus execuções secundárias​

WITH child AS (
  SELECT
    parent_run_id AS stream_run_id,
    count(*) AS child_batches,
    sum(coalesce(rows_read, 0)) AS child_rows_read,
    sum(coalesce(rows_written, 0)) AS child_rows_written,
    sum(coalesce(rows_quarantined, 0)) AS child_rows_quarantined
  FROM main.ops.ctrl_ingestion_runs
  WHERE parent_run_id IS NOT NULL
  GROUP BY parent_run_id
)
SELECT
  s.stream_run_id,
  s.status,
  s.batches_processed,
  c.child_batches,
  s.total_rows_read,
  c.child_rows_read,
  s.total_rows_written,
  c.child_rows_written,
  s.total_rows_quarantined,
  c.child_rows_quarantined
FROM main.ops.ctrl_ingestion_streams s
LEFT JOIN child c
  ON s.stream_run_id = c.stream_run_id
ORDER BY s.started_at_utc DESC;

Estágios lentos a partir da duração dos estágios​

SELECT
  run_id,
  target_table,
  mode,
  duration_seconds,
  get_json_object(stage_durations_json, '$.read') AS read_seconds,
  get_json_object(stage_durations_json, '$.quality') AS quality_seconds,
  get_json_object(stage_durations_json, '$.write') AS write_seconds,
  get_json_object(stage_durations_json, '$.state') AS state_seconds,
  get_json_object(stage_durations_json, '$.lineage') AS lineage_seconds
FROM main.ops.ctrl_ingestion_runs
WHERE run_date >= current_date() - INTERVAL 7 DAYS
ORDER BY duration_seconds DESC
LIMIT 50;

Estimativa de rendimento e custo operacional​

WITH rates AS (
  SELECT 2.0 AS dbu_per_hour, 0.40 AS currency_per_dbu
)
SELECT
  r.target_table,
  r.mode,
  r.source_connector,
  r.rows_read,
  r.rows_written,
  r.duration_seconds,
  CASE WHEN r.duration_seconds > 0 THEN r.rows_written / r.duration_seconds END AS rows_written_per_second,
  (r.duration_seconds / 3600.0) * rates.dbu_per_hour AS estimated_dbu,
  (r.duration_seconds / 3600.0) * rates.dbu_per_hour * rates.currency_per_dbu AS estimated_cost
FROM main.ops.ctrl_ingestion_runs r
CROSS JOIN rates
WHERE r.status = 'SUCCESS'
  AND r.run_date >= current_date() - INTERVAL 30 DAYS
ORDER BY estimated_cost DESC;

Bloqueios ativos ou expirados​

SELECT
  target_table,
  run_id,
  owner,
  status,
  acquired_at_utc,
  expires_at_utc,
  released_at_utc,
  CASE
    WHEN status = 'ACTIVE' AND expires_at_utc < current_timestamp() THEN 'EXPIRED_BUT_NOT_RELEASED'
    ELSE status
  END AS effective_status
FROM main.ops.ctrl_ingestion_locks
ORDER BY acquired_at_utc DESC;

Inventário de versão da estrutura​

SELECT
  framework_version,
  ctrl_schema_version,
  runtime_type,
  spark_version,
  python_version,
  count(*) AS runs
FROM main.ops.ctrl_ingestion_runs
WHERE run_date >= current_date() - INTERVAL 30 DAYS
GROUP BY framework_version, ctrl_schema_version, runtime_type, spark_version, python_version
ORDER BY runs DESC;

Fluxos de trabalho operacionais​

Painel diário de saúde​

Baseie-o em ctrl_ingestion_runs, agrupado por status, target_table, mode, source_connector e runtime_type. Inclui mensagens de erro recentes e links para executar IDs.

Monitoramento de frescor​

Use ctrl_ingestion_state.last_success_at_utc mais ctrl_ingestion_operations.freshness_sla_minutes. Alerta fora do ContractForge; a biblioteca registra os metadados e evidências.

Resposta de qualidade​

Use ctrl_ingestion_quality para resumo em nível de regra e ctrl_ingestion_quarantine para correção em nível de registro. As regras de anulação e as regras de quarentena exigem caminhos de resposta diferentes.

Suporte de runtime​

Comece com ctrl_ingestion_runs.error_message, junte ctrl_ingestion_errors para rastreamento completo e verifique framework_version, runtime_type e metadados do conector antes de alterar o código.

Comandos de manutenção​

Os relatórios de retenção e custos são utilitários operacionais. Os relatórios de custos estimam o custo do runtime a partir de taxas fornecidas pelo usuário; eles não substituem as exportações de faturamento na nuvem.

contractforge maintenance ctrl-retention --catalog main --ctrl-schema ops --retention-days 30
contractforge maintenance cost-report --catalog main --ctrl-schema ops --dbu-per-hour 2 --currency-per-dbu 0.40

Orientação de retenção

Aplique a retenção primeiro a tabelas particionadas por data de alto volume, como execuções e erros. Mantenha o histórico de quarentena e alterações de esquema de acordo com a política de auditoria e correção de qualidade de dados da organização.
