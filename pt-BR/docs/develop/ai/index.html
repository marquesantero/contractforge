ContractForge AI

Esta página é mantida para links mais antigos. A documentação principal agora reside no [área ContractForge AI] dedicada (/docs/ai/overview).

ContractForge AI ajuda as equipes a transformar intenções, esquemas e contexto do projeto em artefatos ContractForge revisáveis. Não é um substituto para o núcleo planejador ou adaptadores. A validação determinística continua sendo a porta de entrada; saída do provedor é consultivo, a menos que passe na validação do esquema local e permaneça dentro do superfície de especificação do projeto permitida.

Modelo operacional​

O pacote AI mantém os limites do ContractForge intactos:

  • contratos mantêm origem, destino, modo de gravação, qualidade, acesso e transformação semantics;
  • os YAMLs de ambiente mantêm o tempo de execução do adaptador, as evidências e as configurações de implantação;
  • pacotes de adaptadores possuem comportamento Databricks, AWS, Snowflake, Fabric e GCP;
  • provedores de modelos não podem mover configurações específicas da plataforma para ingestão contratos.
  • provedores de modelos não podem substituir silenciosamente campos de identidade determinísticos, como como conector, caminho de origem, destino, camada ou modo de gravação. Provedor conflitante as propostas são rejeitadas e registradas na auditoria de propostas do fornecedor.
  • provedores de modelos podem sugerir atualizações de minutas de contrato com suporte, mas os campos que podem alterar o comportamento ou a implantação, como transform, shape, quality_rules, annotations, operations e dab_compute, permanecem sempre revisão necessária. Campos de tempo de execução, segredos e configurações de implantação não suportados são rejeitados e auditados em vez de serem inscritos em contratos.

Gerar um projeto​

Use plan-project quando o prompt ainda estiver difícil e você desejar recomendações commands before writing files:

contractforge-ai plan-project \
  --intent "Create an AWS Glue bronze ingestion project from s3://landing/orders into analytics.bronze.b_orders using append." \
  --schema schemas/orders.json \
  --format markdown

Use generate-project quando a origem, o destino e o destino do adaptador forem explícitos:

contractforge-ai generate-project \
  --target aws-glue-iceberg \
  --schema schemas/orders.json \
  --project-name orders_aws \
  --connector s3 \
  --source-path s3://landing/orders \
  --target-catalog analytics \
  --target-schema bronze \
  --target-table b_orders \
  --output-dir generated/orders-aws

Use guided-project quando um comando precisar planejar e estruturar:

contractforge-ai guided-project \
  --intent "Create a bronze to gold medallion project from s3://landing/orders into analytics.bronze.b_orders using hash_diff_upsert." \
  --schema schemas/orders.json \
  --target contractforge-yaml \
  --allow-review-required \
  --output-dir generated/orders

Os alvos de geração suportados incluem contractforge-yaml, contractforge-python, databricks-dab, aws-glue-iceberg, snowflake-sql-warehouse, fabric-lakehouse, gcp-bigquery, dbt e classic-pyspark.

Consulte geração de projeto de IA para project.yaml canônico, Detalhes de environment.yaml, connection.yaml e destino do adaptador.

Palavras de armazenamento e formato de tabela não são tratadas como escolhas de plataforma por eles mesmos. Por exemplo, s3://... seleciona o conector s3 e iceberg pode descrever um formato de tabela de destino, mas nenhum deles força AWS sozinho. Se o nomes de prompt AWS Glue, Athena ou Lake Formation, AWS torna-se a dica do adaptador. Se o prompt nomear Databricks, Unity Catalog, Asset Bundles ou Auto Loader, Databricks torna-se a dica do adaptador. Serviço Snowflake, Fabric e GCP/BigQuery os nomes são mapeados para seus destinos de adaptador correspondentes. Sem uma plataforma explícita, a IA o planejamento pode recomendar vários caminhos de adaptador para a mesma intenção de contrato. Um alvo de geração explícito também atua como uma dica determinística da plataforma: --target aws-glue-iceberg, --target snowflake-sql-warehouse, --target fabric-lakehouse, --target gcp-bigquery e --target databricks-dab adiciona a estrutura de ambiente correspondente, mesmo quando o o próprio prompt descreve apenas a origem e o destino.

Deriva mínima da plataforma​

Para intenção portátil, ContractForge AI pode gerar uma ingestão canônica contrate e vincule vários adaptadores por meio de project.yaml:

environments:
  databricks: environments/databricks.environment.yaml
  aws: environments/aws.environment.yaml
  snowflake: environments/snowflake.environment.yaml
  fabric: environments/fabric.environment.yaml
  gcp: environments/gcp.environment.yaml

execution_order:
  - name: bronze_orders
    contracts:
      databricks: contracts/bronze/orders.ingestion.yaml
      aws: contracts/bronze/orders.ingestion.yaml
      snowflake: contracts/bronze/orders.ingestion.yaml
      fabric: contracts/bronze/orders.ingestion.yaml
      gcp: contracts/bronze/orders.ingestion.yaml

Quando o destino selecionado é explicitamente aws-glue-iceberg, o gerador grava uma estrutura de ambiente AWS e aponta as etapas do projeto AWS para o mesmo canônico contratos. Isso muda o projeto e o limite de implantação, não o contrato semântica. Use caminhos de contrato específicos do adaptador somente quando extensões de plataforma são obrigatórios e passíveis de revisão.

Validar antes da implantação​

validate-project-structure verifica o formato do projeto, arquivos de ambiente, conexão YAMLs, pacotes de contrato divididos e planejamento de adaptador opcional:

contractforge-ai validate-project-structure generated/orders-aws \
  --adapter aws \
  --format html > project_validation.html

O validador usa ContractForge Core para carregar pacotes e resolver source.type: connection. As configurações de origem no nível de ingestão substituem as reutilizáveis a conexão YAML é padronizada após a conexão ser carregada.

Com --adapter, o pacote AI chama o planejador público do adaptador. SUPPORTED mapeia para READY; a saída do planejador somente de aviso é mapeada para READY_WITH_WARNINGS com ready=true; bloqueadores, semântica sem suporte e descobertas inseguras permanecem non-ready.

READY_WITH_WARNINGS é deliberado. Ele preserva a portabilidade real e avisos em escala de produção sem marcar um projeto em bom estado como inválido. Para revisão, --format html renderiza um relatório rico com cartões legíveis para descobertas e evidências de planejamento do adaptador em vez de tabelas amplas que entram em colapso caminhos longos e códigos de aviso.

Consulte Validação e paridade de IA para a pasta do projeto validação e relatórios de comparação Databricks/AWS.

Provider routing​

Encaminhe o trabalho antes de ligar para um provedor modelo:

contractforge-ai route-task \
  --intent "Summarize low-risk metadata improvements for this connector" \
  --knowledge-index .contractforge-ai/knowledge.json \
  --prefer-http-only \
  --format markdown

--prefer-http-only pode reduzir a pressão de dependência do SDK para aconselhamento de baixo risco tarefas. Não é um rebaixamento de qualidade. Planejamento de projetos, geração de contratos e deployable-artifact workflows still require strict structured-output support porque sua saída pode afetar o comportamento real de ingestão.

As verificações de qualidade do provedor comparam a saída do modelo com o status determinístico. Se a linha de base é NEEDS_DECISIONS, INVALID, UNSAFE, UNSUPPORTED ou FAIL, resumos e recomendações de provedores não devem alegar que o projeto é pronto para implantar, publicar, executar ou prosseguir.

Review artifact​

Os projetos gerados consolidam a revisão em AI_REVIEW.html. Direto os planos do projeto usam PROJECT_REVIEW.html, e a validação completa da pasta do projeto pode emitir project_validation.html através de validate-project-structure --format html. Esses relatórios contêm intenções interpretadas, artefatos gerados, decisões, evidência de contexto, status de validação, evidência do adaptador e provedor orientação quando aplicável. Trate o relatório HTML como superfície de aprovação antes Implantação.

A análise de evidências operacionais está documentada em Inteligência de evidências.
